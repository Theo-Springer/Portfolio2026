CSV Analyst

Un outil qui transforme un fichier CSV brut en analyse statistique exploitable en quelques secondes, sans écrire une seule ligne de code côté utilisateur.

Domaine

Le projet

CSV Analyst comble le vide entre un tableur Excel rigide et un script de data science complexe. L'idée : n'importe qui charge un jeu de données, en comprend la structure statistique et le visualise, sans compétence technique. L'outil calcule tout seul la moyenne, la médiane, les extrêmes et l'écart-type, et repère les valeurs manquantes.

Pour moi, c'était l'occasion de manipuler des données réelles avec Pandas et de construire une interface dynamique en Python pur grâce à Streamlit.

Une interface qui réagit à l'état

La force de Streamlit, c'est son modèle « réactif » : le script se ré-exécute à chaque interaction, et l'affichage se recompose en fonction de l'état courant. Ici, toute l'analyse est conditionnée à la présence d'un fichier.

app.py
import streamlit as st
import pandas as pd
from src.processor import get_stats   # logique de calcul isolée dans son module

st.set_page_config(page_title="CSV Analyst", layout="wide")
st.title("📊 CSV Analyst")

# Import du fichier dans la barre latérale
with st.sidebar:
    uploaded_file = st.file_uploader("Importer un fichier CSV", type="csv")

# Tant qu'aucun fichier n'est chargé, on n'analyse rien
if uploaded_file is not None:
    df = pd.read_csv(uploaded_file)
    st.success(f"Fichier '{uploaded_file.name}' chargé avec succès !")

    # Le curseur pilote directement le nombre de lignes affichées
    rows = st.slider("Nombre de lignes à voir", 5, len(df), 10)
    st.dataframe(df.head(rows), use_container_width=True)

    # Une seule ligne de Pandas produit toutes les stats descriptives
    st.subheader("📈 Statistiques automatiques")
    st.write(df.describe())
else:
    st.warning("Veuillez importer un fichier CSV pour commencer.")
À retenir : Le if uploaded_file is not None est le pivot de l'app : il sépare nettement l'état « en attente » de l'état « analyse ». Tout ce qui suit ne s'exécute que lorsqu'un fichier existe, ce qui évite les erreurs et garde l'interface claire.

Deux détails que je trouve élégants dans ce code court : df.describe() qui, en un appel, sort moyenne, écart-type, quartiles et extrêmes de chaque colonne numérique ; et le st.slider dont la valeur alimente directement df.head(rows), si bien que déplacer le curseur met à jour le tableau instantanément.

Séparer l'affichage du calcul

J'ai volontairement sorti la logique de calcul dans un module src/processor.py, importé en haut du fichier. L'interface (app.py) se concentre sur l'affichage ; le traitement des données vit ailleurs. Cette séparation rend chaque partie testable et réutilisable indépendamment.

Ce que j'en retiens

Ce projet m'a appris à penser « donnée » : lire un CSV, en inspecter la structure, produire des indicateurs pertinents. Il m'a aussi montré qu'on peut livrer une vraie interface web utile sans quitter Python, à condition de bien séparer les responsabilités entre le calcul et la présentation.
