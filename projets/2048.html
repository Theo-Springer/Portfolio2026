Le projet

2048 se joue sur une grille où l'on pousse les tuiles dans quatre directions ; les tuiles de même valeur fusionnent et doublent. Le réflexe naïf serait d'écrire quatre algorithmes séparés, un par flèche. C'est long, répétitif et propice aux bugs.

La bonne idée, c'est de remarquer que les quatre déplacements sont le même mouvement vu sous différents angles. Il suffit d'en coder un seul.

Un seul mouvement, quatre directions

J'implémente vraiment le déplacement « vers la gauche ». Les trois autres s'y ramènent par de simples transformations réversibles de la grille : une symétrie (inverse) ou une transposition (transposer).

logique.py
# Le SEUL vrai déplacement : tasser puis fusionner vers la gauche
def left(game):
    game, done = cover_up(game)          # 1. on tasse les tuiles à gauche
    game, done = fusionner(game, done)   # 2. on fusionne les paires égales
    game = cover_up(game)[0]             # 3. on re-tasse les trous laissés par la fusion
    return game, done

# Vers la droite = miroir horizontal, on joue "à gauche", puis on remet le miroir
def right(game):
    game = inverse(game)
    game, done = cover_up(game)
    game, done = fusionner(game, done)
    game = cover_up(game)[0]
    game = inverse(game)
    return game, done

# Vers le haut = on transpose (colonnes ↔ lignes), on joue "à gauche", on transpose
def up(game):
    game = transposer(game)
    game, done = cover_up(game)
    game, done = fusionner(game, done)
    game = cover_up(game)[0]
    game = transposer(game)
    return game, done

# Vers le bas = transposition + miroir combinés
def down(game):
    game = inverse(transposer(game))
    game, done = cover_up(game)
    game, done = fusionner(game, done)
    game = cover_up(game)[0]
    game = transposer(inverse(game))
    return game, done
L'idée clé : Les quatre fonctions partagent exactement le même cœur (tasser → fusionner → tasser). Seule change la transformation qu'on applique avant et qu'on annule après. On réoriente la grille pour que le problème redevienne toujours « pousser vers la gauche ».

La fusion, cœur du jeu

Une fois les tuiles tassées à gauche, il reste à fusionner les voisines identiques. La règle du 2048 : on parcourt de gauche à droite, et deux tuiles égales et non nulles fusionnent en doublant la première.

logique.py
def fusionner(mat, done):
    for i in range(c.GRID_LEN):
        for j in range(c.GRID_LEN - 1):
            # deux voisines identiques et non nulles ne fusionnent qu'une fois
            if mat[i][j] == mat[i][j + 1] and mat[i][j] != 0:
                mat[i][j] *= 2          # la gauche double
                mat[i][j + 1] = 0       # la droite se vide
                done = True             # un mouvement a bien eu lieu
    return mat, done
À retenir : Le drapeau done qui remonte de fonction en fonction n'est pas décoratif : il indique si le coup a réellement modifié la grille. C'est lui qui décide s'il faut faire apparaître une nouvelle tuile, car en 2048 une tuile n'apparaît qu'après un mouvement valide.

Ce que j'en retiens

Ce projet, réalisé en fin de Terminale, est celui qui m'a fait aimer l'algorithmique. Il montre qu'un bon raisonnement en amont, ici factoriser quatre problèmes en un seul, vaut mieux que du code répété. Manipuler des matrices et faire circuler un état entre les fonctions : ces réflexes me servent encore.
