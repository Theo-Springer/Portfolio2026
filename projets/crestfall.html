Crestfall

Un jeu de rôle en console avec cinq classes jouables, un système de combat, un inventaire et un butin à raretés. Le projet où j'ai vraiment maîtrisé la programmation orientée objet.

Le projet

Crestfall est un RPG jouable au clavier. Le joueur choisit une classe (Machiniste, Pistolero, Spectre, Briseur ou Archaniste), puis enchaîne des combats au tour par tour, gagne de l'expérience, ramasse des objets et affronte des ennemis générés selon la zone.

Un jeu de ce type est un terrain d'entraînement parfait pour la POO. Un ennemi et un joueur partagent les mêmes bases (des points de vie, une attaque, une défense) mais se comportent différemment. C'est exactement ce que l'héritage et le polymorphisme servent à modéliser.

Une base commune : la classe Entite

Toutes les créatures du jeu, joueur comme ennemis, héritent d'une classe Entite. Elle regroupe les statistiques et les comportements partagés. Voici la montée de niveau, qui gère proprement le cas où un gros gain d'expérience fait grimper plusieurs niveaux d'un coup.

src/entites/Entite.java
public void gagnerExperience(int exp) {
    this.experience += exp;

    // Une seule victoire peut faire monter PLUSIEURS niveaux : d'où le while
    while (this.experience >= this.expMax) {
        this.experience -= this.expMax;
        this.expMax += 50;      // chaque palier coûte plus cher que le précédent
        this.niveau++;
        this.hpMax   += 10;     // les stats progressent à chaque niveau
        this.attaque += 2;
        this.defense += 1;
        this.hp = this.hpMax;   // soin complet en récompense
        System.out.println(this.nom + " a atteint le niveau " + this.niveau + " !");
    }
}
À retenir : Le while plutôt qu'un simple if est un choix conscient : si l'ennemi rapporte 300 XP et qu'il en faut 100 par palier, la boucle enchaîne les niveaux tant que le seuil est franchi. Un if aurait « perdu » l'excédent.

Le polymorphisme en action

Chaque classe jouable a ses propres capacités spéciales. Le moteur de combat n'a pas besoin de les connaître à l'avance : il interroge le type réel du joueur au moment d'agir et déclenche la bonne capacité.

src/Combat.java
// Le combat ne code pas "en dur" une classe : il s'adapte au personnage joué
private void attaqueSpeciale1() {
    if      (joueur instanceof Machiniste) ((Machiniste) joueur).TourelleAutonome(ennemi);
    else if (joueur instanceof Pistolero)  ((Pistolero)  joueur).TirDePrecision(ennemi);
    else if (joueur instanceof Spectre)    ((Spectre)    joueur).FrappeDansLombre(ennemi, tour);
    else if (joueur instanceof Briseur)    ((Briseur)    joueur).FrappePhenomenale(ennemi);
    else if (joueur instanceof Archaniste) ((Archaniste) joueur).DechargeArchanique(ennemi);
    else System.out.println("  Cette entité n'a pas d'attaque spéciale 1.");
}
Le raisonnement : Chaque classe (Machiniste, Pistolero…) étend Entite et ajoute ses propres méthodes. Grâce à instanceof, un même bouton « Spéciale 1 » déclenche une action totalement différente selon le personnage. Ajouter une nouvelle classe ne casse rien : il suffit d'une branche de plus.

Enums et streams pour le butin

Les objets ont une rareté, et chaque rareté porte son propre taux d'apparition. J'ai modélisé ça avec un enum, puis filtré le butin possible avec l'API Stream de Java.

src/Combat.java
private void genererLoot(Inventaire inventaire) {
    double roll = Math.random();

    // La rareté est tirée selon les taux portés par l'enum lui-même
    Objet.Rarete rarete;
    if      (roll < Objet.Rarete.EPIQUE.taux)                               rarete = Objet.Rarete.EPIQUE;
    else if (roll < Objet.Rarete.RARE.taux + Objet.Rarete.EPIQUE.taux)      rarete = Objet.Rarete.RARE;
    else                                                                    rarete = Objet.Rarete.COMMUN;

    // On ne garde que les objets de la rareté tirée, puis on en choisit un au hasard
    List<Objet> possibles = ObjetCree.OBJET.values().stream()
            .filter(o -> o.rarete == rarete)
            .collect(Collectors.toList());

    Objet drop = possibles.get(new Random().nextInt(possibles.size()));
    inventaire.ajouter(drop.id);
}
À retenir : Stocker le taux de drop dans l'enum garde la logique au bon endroit : le jour où j'équilibre le jeu, je change une valeur, pas dix conditions. Le stream().filter().collect() exprime « donne-moi les objets de cette rareté » en une ligne lisible.

Ce que j'en retiens

Crestfall a fait passer la POO de la théorie à la pratique. Héritage pour factoriser le commun, polymorphisme pour spécialiser le comportement, enums pour donner du sens aux données, streams pour manipuler des collections proprement. C'est aussi le projet qui m'a appris à découper un programme en fichiers cohérents plutôt qu'en un seul bloc.
