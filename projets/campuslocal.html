Le projet

CampusLocal est mon projet le plus ambitieux en volume. C'est un réseau social complet organisé en monorepo : un seul dépôt qui contient un backend en FastAPI, un frontend « officiel » et un frontend « démo » autonome, le tout géré par les workspaces npm.

Je ne l'ai jamais déployé en production : c'était un banc d'essai personnel pour explorer une architecture full-stack complète, aujourd'hui en pause. Ce que je montre ici, c'est donc la conception et le code, pas un produit fini.

Écrire trois services, ce n'est que la moitié du travail. Les faire démarrer ensemble d'une seule commande, sur Windows comme sur Mac ou Linux, sans que celui qui clone le dépôt ait à lire trois pages de documentation : voilà le vrai défi. C'est le rôle de mon script de lancement.

Un lanceur qui parle à toutes les machines

Avant de démarrer quoi que ce soit, le script vérifie que l'environnement est prêt. Il tolère les différences entre systèmes : par exemple, Python s'appelle python sur Windows mais souvent python3 sur Mac et Linux.

launch.js
function checkPrerequisites() {
  logSection('🔍 Vérification des prérequis');

  // Node.js est indispensable : sans lui, on arrête tout de suite
  try {
    const nodeVersion = execSync('node --version').toString().trim();
    log(`✅ Node.js: ${nodeVersion}`, 'green');
  } catch (e) {
    log('❌ Node.js introuvable !', 'red');
    process.exit(1);
  }

  // Python : on tente 'python', puis on retombe sur 'python3'
  try {
    const py = execSync('python --version').toString().trim();
    log(`✅ Python: ${py}`, 'green');
  } catch (e) {
    try {
      const py = execSync('python3 --version').toString().trim();
      log(`✅ Python: ${py}`, 'green');
    } catch (e2) {
      log('⚠️  Python introuvable (nécessaire au backend)', 'yellow');
    }
  }
}
À retenir : Un outil de développement doit être robuste face à l'environnement de celui qui l'utilise. Le double try/catch imbriqué gère élégamment le cas « python vs python3 » sans supposer sur quel OS on se trouve.

Trois services, un seul Ctrl+C

Une fois les prérequis validés, le script lance les trois services en parallèle et surveille chacun. Il affiche ensuite un tableau de bord clair des URL disponibles.

launch.js
function startService(name, command, cwd, port) {
  return new Promise((resolve) => {
    log(`🚀 Démarrage de ${name} sur le port ${port}...`, 'yellow');

    const service = spawn('npm', ['run', command], { cwd, shell: true, stdio: 'inherit' });

    service.on('error', (err) => { log(`❌ ${name}: ${err.message}`, 'red'); resolve(); });
    service.on('exit',  (code) => { if (code !== 0) log(`❌ ${name} a quitté (code ${code})`, 'red'); resolve(); });
  });
}

// Les 3 services démarrent ensemble, sans se bloquer les uns les autres
services.forEach(service => startService(service.name, service.command, service.dir, service.port));

// Un seul Ctrl+C arrête proprement tout l'ensemble
process.on('SIGINT', () => {
  log('\n👋 Arrêt de tous les services...', 'cyan');
  process.exit(0);
});
Le raisonnement : Chaque service est lancé dans un processus enfant (spawn) enveloppé dans une Promise, ce qui permet de tous les démarrer sans attendre. L'écoute de SIGINT remplace « je ferme mes trois terminaux à la main » par un simple Ctrl+C.

Ce que j'en retiens

CampusLocal m'a confronté à l'organisation d'un vrai projet à plusieurs briques. J'ai appris à structurer un monorepo, à faire cohabiter un backend Python et des frontends JavaScript, et à rendre le projet simple à lancer pour qui le découvre. Le frontend démo tourne même sans backend, pour qu'on puisse le tester en dix secondes.
